## Complete Workflow Overview

The Revize order editing workflow is designed to be seamless for customers while maintaining business control and integration with your existing systems.

<Steps>
<Step title="Order Placed">
Customer completes checkout on your Shopify store.
- Order is created in Shopify
- Revize automatically puts order "on hold"
- Edit window begins (configurable duration)
</Step>

<Step title="Customer Notification">
Customer receives order confirmation with editing options.
- Email includes edit link and time limit
- Thank you page shows edit button
- Customer account displays edit option
</Step>

<Step title="Editing Window Active">
Customer can access and modify their order.
- Real-time inventory checking
- Automatic payment processing
- Live shipping calculations
- Smart upsell suggestions
</Step>

<Step title="Changes Applied">
All modifications are processed automatically.
- Payment adjustments completed
- Inventory levels updated
- New confirmation email sent
- 3PL systems notified
</Step>

<Step title="Edit Window Closes">
After the time limit, editing is disabled.
- Order released from hold
- Fulfillment process begins
- Standard Shopify workflow resumes
</Step>
</Steps>

- Timer starts for edit window

```mermaid
graph TD
    A[Customer Accesses Edit] → B[Load Current Order]
    B → C[Display Edit Options]
    C → D[Customer Makes Changes]
    D → E[Validate Changes]
    E → F[Process Payments]
    F → G[Update Order]
    G → H[Send Confirmation]
```

**What happens:**
- Real-time validation of changes
- Automatic payment processing
- Inventory reservation updates
- Order modifications applied
</Tab>

<Tab title="Completion">
```mermaid
graph TD
    A[Edit Window Expires] → B[Remove Hold Status]
    B → C[Release to Fulfillment]
    C → D[Notify 3PL Partners]
    D → E[Standard Processing]
```

**What happens:**
- Order released from editing
- Fulfillment partners notified
- Normal shipping process begins
- Tracking information generated
</Tab>

## Business Logic & Rules

### Inventory Management

<Warning>
Inventory is reserved during the editing window to prevent overselling.
</Warning>

**Reservation Process:**
1. Items reserved when order placed
2. Additional reservations for increases
3. Releases for quantity decreases
4. Auto-release when window expires

### Payment Processing

<Tabs>
<Tab title="Price Increases">
**When order total increases:**
- Charge difference to original payment method
- Backup payment methods if primary fails
- Customer notification for payment issues
- Order hold if payment cannot be processed
</Tab>

<Tab title="Price Decreases">
**When order total decreases:**
- Automatic refund to original payment method
- Refund processing within 3-5 business days
- Email confirmation of refund amount
- Updated receipt provided
</Tab>

<Tab title="Payment Failures">
**If payment processing fails:**
- Customer immediately notified
- Alternative payment options provided
- Order held until payment resolved
- Support team automatically alerted
</Tab>
</Tabs>

### Shipping Calculations

**Real-time updates for:**
- Address changes affecting zones
- Weight/size changes from products
- Shipping method upgrades/downgrades
- Express shipping availability
- Free shipping threshold impacts
