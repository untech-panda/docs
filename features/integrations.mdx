## Popular 3PL Integrations

### ShipBob

<AccordionGroup>
<Accordion title="Setup & Configuration" icon="cog">
**Integration Process:**
1. Connect your ShipBob account in Revize settings
2. Map product SKUs between systems
3. Configure inventory sync preferences
4. Set fulfillment hold/release triggers

**Required Information:**
- ShipBob API credentials
- Warehouse location preferences
- Product mapping spreadsheet
- Shipping service preferences
</Accordion>

<Accordion title="Order Flow" icon="arrow-right">
**How it works:**
1. **Order Placed**: Revize notifies ShipBob to hold fulfillment
2. **Edit Window**: Orders remain on hold during editing period
3. **Changes Made**: ShipBob receives updated order details instantly
4. **Window Expires**: Fulfillment automatically resumes
5. **Shipping**: Normal ShipBob process continues

**Real-time updates:**
- Inventory levels synchronized
- Shipping costs recalculated
- Warehouse assignments updated
- Priority shipping adjusted
</Accordion>

<Accordion title="Advanced Features" icon="star">
**Premium capabilities:**
- **Multi-warehouse optimization**: Revize suggests best fulfillment location
- **Inventory reservation**: Items held during edit windows
- **Bulk order handling**: Special processing for large orders
- **Custom packaging**: Modifications sync to packaging preferences
</Accordion>
</AccordionGroup>

### Fulfillment by Amazon (FBA)

<AccordionGroup>
<Accordion title="FBA Integration" icon="amazon">
**Special considerations for FBA:**
- Limited modification capabilities due to Amazon's system
- Address changes supported for unshipped orders
- Quantity changes may require new shipments
- Returns processed through Amazon's system

**Supported modifications:**
- Shipping address updates (before shipment)
- Order cancellations (full refund)
- Some quantity adjustments
- Gift message changes
</Accordion>

<Accordion title="Hybrid Fulfillment" icon="balance-scale">
**Best practices for mixed fulfillment:**
- FBA items: Limited editing capability
- 3PL items: Full editing features
- Dropship items: Vendor-dependent capabilities
- In-house items: Complete flexibility

**Order splitting:**
- Automatic detection of fulfillment methods
- Separate edit windows per fulfillment type
- Combined shipping optimization
- Unified customer experience
</Accordion>
</AccordionGroup>

### REST API

<AccordionGroup>
<Accordion title="API Endpoints" icon="terminal">
**Available endpoints:**
- `GET /orders/{id}/edits` - Retrieve edit history
- `POST /orders/{id}/edit` - Programmatic order editing
- `GET /orders/{id}/edit-window` - Check edit availability
- `PUT /orders/{id}/extend-window` - Extend edit time
- `GET /analytics/edits` - Edit performance data
</Accordion>

<Accordion title="Authentication" icon="key">
**Security methods:**
- API key authentication
- OAuth 2.0 flow
- JWT tokens for session management
- IP whitelisting options
- Rate limiting protection
</Accordion>
</AccordionGroup>

## Integration Best Practices

<AccordionGroup>
<Accordion title="Testing Strategy" icon="flask">
**Comprehensive testing approach:**
1. **Sandbox testing**: Use test environments first
2. **Staged rollout**: Gradually enable integrations
3. **Monitor closely**: Watch for errors and performance issues
4. **Fallback plans**: Prepare manual processes for outages
5. **Regular validation**: Periodic end-to-end testing
</Accordion>

<Accordion title="Performance Optimization" icon="tachometer-alt">
**Optimization techniques:**
- **Batch processing**: Group updates when possible
- **Caching**: Store frequently accessed data locally
- **Async operations**: Use background processing for non-critical updates
- **Rate limiting**: Respect partner API limits
- **Error handling**: Graceful degradation during issues
</Accordion>

<Accordion title="Monitoring & Alerts" icon="bell">
**Proactive monitoring:**
- Integration health checks
- Performance metrics tracking
- Error rate monitoring
- Data consistency validation
- Automated alert system
- Regular audit reports
</Accordion>
</AccordionGroup>
