# Test Order Editing Setup

Verify your Revize configuration works correctly with these safe testing methods.

## Testing with Draft Orders

<Steps>
  <Step title="Create a Draft Order">
    **Set up a test order in Shopify Admin**
    
    1. Go to **Orders** → **Create order** in your Shopify admin
    2. Add yourself as the customer (use your email)
    3. Add 2-3 different products to test editing options
    4. Set shipping address and method
    5. **Save as draft** (don't process payment yet)
  </Step>
  
  <Step title="Convert to Real Order">
    **Process the draft order**
    
    1. Click **Send invoice** to customer (yourself)
    2. Complete the test purchase using a test payment method
    3. Check that the order appears in your orders list
    4. Verify the order status shows as "Paid"
  </Step>
  
  <Step title="Test Customer Editing">
    **Access the order as a customer would**
    
    1. Check your email for the order confirmation
    2. Click the **Order Status** link in the email
    3. Look for the Revize editing blocks you configured
    4. Test editing features: address changes, quantity adjustments, etc.
    5. Verify changes save correctly and reflect in Shopify admin
  </Step>
</Steps>

## What to Test

<AccordionGroup>
  <Accordion title="Core Editing Functions" icon="edit">
    **Test these essential features:**
    
    - ✅ **Address editing**: Change shipping address, verify validation
    - ✅ **Quantity changes**: Increase/decrease product quantities
    - ✅ **Variant swapping**: Change size, color, or style options

    - ✅ **Order cancellation**: Cancel order and verify refund process

    - ✅ **Within time limit**: Editing options are available
    - ✅ **After deadline**: Editing becomes unavailable
    - ✅ **Order processing**: Orders release to fulfillment correctly
    - ✅ **Hold behavior**: Orders pause/resume as configured
  </Accordion>
  
  <Accordion title="Email & Notifications" icon="envelope">
    **Check customer communication:**
    
    - ✅ **Order confirmation**: Contains edit links and instructions
    - ✅ **Edit confirmations**: Customers receive update notifications
    - ✅ **Deadline reminders**: Time-based notifications (if enabled)
    - ✅ **Support routing**: Test tickets reach the right email
  </Accordion>
  
  <Accordion title="App Block Display" icon="layout">
    **Verify theme integration:**
    
    - ✅ **Order Status page**: Customer portal appears correctly
    - ✅ **Thank You page**: Edit message displays properly
    - ✅ **Mobile responsiveness**: Works on all device sizes
    - ✅ **Theme compatibility**: Matches your store's design
  </Accordion>
</AccordionGroup>

## Safe Testing Environment

<Warning>
Use these methods to test safely without affecting real customers or inventory.
</Warning>

### Test Customer Setup
- **Use your own email** for test orders
- **Create a test customer account** in Shopify admin
- **Use Shopify's test payment methods** (no real charges)
- **Test during off-peak hours** to avoid customer confusion

### Inventory Protection
- **Use products with high inventory** for quantity testing
- **Create test-only products** if needed
- **Monitor inventory levels** during testing
- **Restore inventory** after testing if needed

## Common Testing Issues

<AccordionGroup>
  <Accordion title="Editing Options Not Appearing" icon="eye-slash">
    **Possible causes:**
    
    - App not enabled in settings
    - Time window already expired
    - App blocks not properly configured
    - Theme compatibility issues
    
    **Solution:** Check app status, time settings, and theme configuration
  </Accordion>
  
  <Accordion title="Changes Not Saving" icon="exclamation-triangle">
    **Possible causes:**
    
    - Internet connectivity issues
    - Browser compatibility problems
    - Insufficient product inventory
    - Payment processing conflicts
    
    **Solution:** Test in different browsers, check inventory levels
  </Accordion>
  
  <Accordion title="Orders Not Releasing" icon="pause">
    **Possible causes:**
    
    - Order hold settings misconfigured
    - Time window still active
    - Manual release required
    - 3PL integration issues
    
    **Solution:** Review hold settings and time configurations
  </Accordion>
</AccordionGroup>
